Introducción

El DOM (Document Object Model en español Modelo de Objetos del Documento) es una API definida para representar e interactuar con cualquier documento HTML o XML. El DOM es un modelo de documento que se carga en el navegador web y que representa el documento como un árbol de nodos, en donde cada nodo representa una parte del documento (puede tratarse de un elemento, una cadena de texto o un comentario). El DOM es una de las APIs más usadas en la Web, pues permite ejecutar código en el navegador para acceder e interactuar con cualquier nodo del documento. Estos nodos pueden crearse, moverse o modificarse. Pueden añadirse a estos nodos manejadores de eventos (event listeners en inglés) que se ejecutarán/activarán cuando ocurra el evento indicado en este manejador. El DOM surgió a partir de la implementación de JavaScript en los navegadores. A esta primera versión también se la conoce como DOM 0 o "Legacy DOM". Hoy en día el grupo WHATWG es el encargado de actualizar el estándar de DOM.

El DOM no es un lenguaje de programación, pero sin él, el lenguaje JavaScript no tendría ningún modelo o noción de páginas web, documentos HTML, documentos XML y sus componentes. El documento en su conjunto, el encabezado, las tablas dentro del documento, los encabezados de tabla, el texto dentro de las celdas de la tabla y todos los demás elementos de un documento son partes del modelo de objetos de documento para ese documento. Todos ellos pueden ser accedidos y manipulados usando el DOM y un lenguaje de scripting como JavaScript. El DOM no es parte del lenguaje JavaScript, sino que es una API web utilizada para crear sitios web. JavaScript también se puede utilizar en otros contextos. Por ejemplo, Node.js ejecuta programas JavaScript en un equipo, pero proporciona un conjunto diferente de API, y la API DOM no es una parte central del tiempo de ejecución .js Node.

No tiene que hacer nada especial para comenzar a usar el DOM. La API se utiliza directamente en JavaScript desde lo que se llama un script, un programa ejecutado por un navegador. Al crear un script, ya sea en línea en un elemento o incluido en la página web, puede comenzar inmediatamente a utilizar la API para que el documento o los objetos de ventana manipulen el documento en sí, o cualquiera de los diversos elementos de la página web (los elementos descendientes del documento).

Localización de elementos DOM mediante selectores

La API selectores proporciona métodos que hacen que sea rápido y fácil recuperar los nodos element del DOM al coincidir con un conjunto de selectores. Esto es mucho más rápido que las técnicas anteriores, en las que era necesario, por ejemplo, usar un bucle en el código JavaScript para localizar los elementos específicos que necesitaba encontrar.

Esta especificación agrega dos métodos nuevos a cualquier objeto que implemente las interfaces Document, DocumentFragment o Element.

querySelector()

Devuelve el primer nodo Element coincidente dentro del subárbol del nodo. Si no se encuentra ningún nodo coincidente, se devuelve null.

querySelectorAll()

Devuelve un NodeList que contiene todos los nodos coincidentes dentro del subárbol del nodo, o un vacío si no se encuentran coincidencias.

Selectores

Los métodos de selección aceptan selectores para determinar qué elemento o elementos deben devolverse. Esto incluye listas de selectores para que pueda agrupar varios selectores en una sola consulta. Para proteger la privacidad del usuario, algunas pseudoclases no son compatibles o se comportan de manera diferente. Por ejemplo, :visited no devolverá coincidencias y :link se trata como :any-link. Solo se pueden seleccionar elementos, por lo que no se admiten pseudoelementos.

Ejemplos

Para seleccionar todos los elementos de párrafo (p) de un documento cuyas clases incluyen warning o note, puede hacer lo siguiente:

const special = document.querySelectorAll("p.warning, p.note");

También puede consultar por ID. Por ejemplo:

const el = document.querySelector("#main, #basic, #exclamation");
Eventos

En el caso de la Web, los eventos se activan dentro de la ventana del navegador y tienden a adjuntarse a un elemento específico que reside en ella: esto puede ser un solo elemento, un conjunto de elementos, el documento HTML cargado en la pestaña actual o toda la ventana del navegador. Hay muchos tipos diferentes de eventos que pueden ocurrir. Por ejemplo:

  • El usuario selecciona un determinado elemento o coloca el cursor sobre un determinado elemento.
  • El usuario elige una tecla en el teclado.
  • El usuario redimensiona o cierra la ventana del navegador.
  • Una página web termina de cargarse.
  • Se envía un formulario.
  • Un vídeo se reproduce, se pausa o finaliza.
  • Se produce un error.

Puede deducir de esto que hay muchos eventos a los que se puede responder.

Cada evento disponible tiene un controlador de eventos,que es un bloque de código (generalmente una función de JavaScript que usted como programador crea) que se ejecuta cuando se dispara el evento. Cuando se define un bloque de código de este tipo para ejecutarse en respuesta a un evento, decimos que estamos registrando un controlador de eventos. Nota: Los controladores de eventos a veces se llaman oyentes de eventos: son prácticamente intercambiables para nuestros propósitos, aunque estrictamente hablando, funcionan juntos. El agente de escucha, escucha el evento que ocurre y el controlador es el código que se ejecuta en respuesta a que ocurra.

Ejemplos

Controladores de eventos en línea: no los use

<button onclick="bgChange()"> Press me </button> function bgChange() { const rndCol = 'rgb(' + random(255) + ',' + random(255) + ',' + random(255) + ')'; document.body.style.backgroundColor = rndCol; }

Controladores de eventos semanticos

<button> Press me </button> const btn = document.querySelector('button'); btn.onclick = function() { const rndCol = 'rgb(' + random(255) + ',' + random(255) + ',' + random(255) + ')'; document.body.style.backgroundColor = rndCol; }

Controladores de eventos multiples

El mecanismo moderno para agregar controladores de eventos es el método addEventListener(). Usándolo, podríamos reescribir nuestro ejemplo de color aleatorio para que se vea así:

<button> Press me </button> const btn = document.querySelector('button'); function bgChange() { const rndCol = 'rgb(' + random(255) + ',' + random(255) + ',' + random(255) + ')'; document.body.style.backgroundColor = rndCol; } btn.addEventListener('click', bgChange);
Interfaz Document

La interfaz document representa cualquier página web cargada en el navegador y sirve como punto de entrada al contenido de la página web, que es el árbol DOM. El árbol DOM incluye elementos como cabeza y cuerpo,entre muchos otros. Proporciona funcionalidad global al documento, como obtener la URL de la página y crear nuevos elementos en el documento.

La interfaz describe las propiedades y métodos comunes para cualquier tipo de documento. Dependiendo del tipo de documento (por ejemplo, HTML, XML,SVG, ...), una API más grande está disponible: los documentos HTML, servidos con el tipo de contenido, también implementan la interfaz HTMLDocument, mientras que los documentos XML y SVG implementan la interfaz XMLDocument.

Constructor

Document(): Crea un nuevo objeto Document.

Propiedades

Document hereda propiedades de su interfaz principal, Node, y por extensión, del elemento primario de esa interfaz, EventTarget.

Document.activeElement: Devuelve el elemento que actualmente tiene foco.

Document.body: Devuelve el body del documento actual.

Document.characterSet: Devuelve el juego de caracteres que utiliza el documento.

Document.childElementCount: Devuelve el número de elementos secundarios del documento actual.

Document.children: Devuelve los elementos secundarios del documento actual.

Document.doctype: Devuelve la definición de tipo de documento (DTD) del documento actual.

Document.documentElement: Devuelve el elemento que es un elemento secundario directo del documento. Para los documentos HTML, normalmente es el objeto HTML que representa la etiqueta html del documento.

Document.documentURI: Devuelve la ubicación del documento como una cadena.

Document.embeds: Devuelve una lista de los elementos incrustados en el documento actual.

Document.firstElementChild: Devuelve el primer elemento secundario del documento actual.

Document.fonts: Devuelve la interfaz FontFaceSet del documento actual.

Document.forms: Devuelve una lista de los elementos form del documento actual.

Document.fullscreenElement: El elemento que se encuentra actualmente en modo de pantalla completa para este documento.

Document.head: Devuelve el elemento head del documento actual.

Document.hidden: Devuelve un valor booleano que indica si la página se considera oculta o no.

Document.images: Devuelve una lista de las imágenes del documento actual.

Document.lastElementChild: Devuelve el último elemento secundario del documento actual.

Document.links: Devuelve una lista de todos los hipervínculos del documento.

Document.plugins: Devuelve una lista de los plugins disponibles.

Document.scripts: Devuelve todos los elementos script del documento.

Document.styleSheets: Devuelve una StyleSheetList de objetos CSSStyleSheet para hojas de estilos vinculadas explícitamente o incrustadas en un documento.

Controladores de eventos

La interfaz document se amplía con controladores de eventos adicionales definidos en GlobalEventHandlers.

Document.onfullscreenchange: Es un controlador de eventos que representa el código al que se va a llamar cuando se genera el evento fullscreenchange.

Document.onfullscreenerror: Es un controlador de eventos que representa el código al que se va a llamar cuando se genera el evento fullscreenerror.

Document.onvisibilitychange: Es un controlador de eventos que representa el código al que se va a llamar cuando se genera el evento visibilitychange.

Métodos

Al igual que las propiedades Document hereda metodos de su interfaz principal, Node, y por extensión, del elemento primario de esa interfaz, EventTarget.

Document.adoptNode(): Adoptar nodo de un documento externo.

Document.append(): Inserta un conjunto de objetos Node u objetos DOMString después del último elemento secundario del documento.

Document.createAttribute(): Crea un nuevo objeto Attr y lo devuelve.

Document.createCDATASection(): Crea un nuevo nodo CDATA y lo devuelve.

Document.createComment(): Crea un nuevo nodo de comentario y lo devuelve.

Document.createDocumentFragment(): Crea un nuevo fragmento de documento.

Document.createElement(): Crea un nuevo elemento con el nombre de etiqueta dado.

Document.createEvent(): Crea un objeto de evento.

Document.createTextNode(): Crea un nodo de texto.

Document.getAnimations(): Devuelve una matriz de todos los objetos Animation actualmente en vigor, cuyos elementos de destino son descendientes del archivo.

Document.getElementById(): Devuelve una referencia de objeto al elemento identificado.

Document.getElementsByClassName(): Devuelve una lista de elementos con el nombre de clase dado.

Document.getElementsByTagName(): Devuelve una lista de elementos con el nombre de etiqueta dado.

Document.getSelection(): Devuelve un objeto Selection que representa el intervalo de texto seleccionado por el usuario o la posición actual del intervalo de instrucciones.

Document.importNode(): Devuelve un clon de un nodo de un documento externo.

Document.prepend(): Inserta un conjunto de objetos Node u objetos DOMString antes del primer elemento secundario del documento.

Document.querySelector(): Devuelve el primer nodo Elemento del documento, en orden de documento, que coincide con los selectores especificados.

Document.querySelectorAll(): Devuelve una lista de todos los nodos Element del documento que coinciden con los selectores especificados.

Document.replaceChildren(): Reemplaza los elementos hijos existentes de un documento por un nuevo conjunto especificado de elementos hijos.

Document.requestStorageAccess(): Devuelve una promesa que se resuelve si se concedió el acceso al almacenamiento de origen y se rechaza si se denegó el acceso.

Eventos

A continuación se nombraran algunos de los eventos mas usados

scroll: Se activa cuando se ha desplazado la vista del documento o un elemento.

select: Se activa cuando se ha seleccionado algún texto.

wheel: Se activa cuando el usuario gira un botón de rueda en un dispositivo señalador (normalmente un ratón).

securitypolicyviolation: Se activa cuando se infringe una política de seguridad de contenido.

Eventos de animación

animationcancel: Se activa cuando una animación aborta inesperadamente.

animationend: Se activa cuando una animación se ha completado normalmente.

animationiteration: Se activa cuando se ha completado una iteración de animación.

animationstart: Se activa cuando se inicia una animación.

Eventos del portapapeles

copy: Se activa cuando el usuario inicia una acción de copia a través de la interfaz de usuario del navegador.

cut: Se activa cuando el usuario inicia una acción de corte a través de la interfaz de usuario del navegador.

paste Se activa cuando el usuario inicia una acción de pegado a través de la interfaz de usuario del navegador.

Eventos de arrastrar y soltar

drag: Se dispara cada pocos cientos de milisegundos mientras el usuario arrastra un elemento o una selección de texto.

dragend: Se activa cuando finaliza una operación de arrastre (soltando un botón del ratón o pulsando la tecla de escape).

dragenter: Se activa cuando un elemento arrastrado o una selección de texto entra en un destino de succión válido.

dragleave: Se activa cuando un elemento arrastrado o una selección de texto deja un destino de entrega válido.

dragover: Se activa cuando se arrastra un elemento o una selección de texto sobre un destino de entrega válido (cada pocos cientos de milisegundos).

dragstart: Se activa cuando el usuario comienza a arrastrar un elemento o una selección de texto.

drop: Se activa cuando se deja caer un elemento o una selección de texto en un destino de entrega válido.

Eventos a pantalla completa

fullscreenchange: Se activa cuando las transiciones entran o salen del modo de pantalla completa.

fullscreenerror: Se activa si se produce un error al intentar entrar o salir del modo de pantalla completa.

Eventos del teclado

keydown: Se activa cuando se presiona una tecla.

keypress: Se activa cuando se presiona hacia abajo una tecla que produce un valor de carácter.

keyup: Se activa cuando se suelta una clave.

Eventos del raton

auxclick: Se activa cuando se ha presionado y liberado un botón de dispositivo señalador no primario (por ejemplo, cualquier botón del mouse que no sea el botón izquierdo) en un elemento.

click: Se activa cuando se presiona y suelta un botón del dispositivo señalador (por ejemplo, el botón principal de un mouse) en un solo elemento.

dblclick: Se activa cuando se hace clic dos veces en un botón de dispositivo señalador (por ejemplo, el botón principal de un mouse) en un solo elemento.

mousedown: Se activa cuando se presiona un botón de dispositivo señalador en un elemento.

mouseenter: Se activa cuando un dispositivo señalador (generalmente un mouse) se mueve sobre el elemento que tiene el oyente conectado.

mouseleave: Se activa cuando el puntero de un dispositivo señalador (generalmente un mouse) se mueve fuera de un elemento que tiene el oyente conectado a él.

mousemove: Se activa cuando un dispositivo señalador (generalmente un mouse) se mueve mientras está sobre un elemento.

mouseout: Se activa cuando un dispositivo señalador (generalmente un mouse) se mueve fuera del elemento al que está conectado el oyente o de uno de sus hijos.

mouseover: Se activa cuando un dispositivo señalador se mueve sobre el elemento al que está conectado el oyente o sobre uno de sus hijos.

mouseup: Se activa cuando se suelta un botón de dispositivo señalador en un elemento.

webkitmouseforcechanged: Se dispara cada vez que la cantidad de presión cambia en la pantalla táctil del trackpad.

webkitmouseforcedown: Se dispara después del evento de ratón hacia abajo tan pronto como se haya aplicado suficiente presión para calificar como un "clic de fuerza".

webkitmouseforcewillbegin: Se ha disparado antes del evento mousedown.

webkitmouseforceup: Se dispara después del evento webkitmouseforcedown tan pronto como la presión se ha reducido lo suficiente como para finalizar el "clic de fuerza".

Eventos de carga y descarga

DOMContentLoaded: Se activa cuando el documento se ha cargado y analizado por completo, sin esperar a que las hojas de estilo, las imágenes y los subtramos terminen de cargarse.

readystatechange: Se activa cuando el atributo readyState de un documento ha cambiado.

Eventos de puntero

gotpointercapture: Se activa cuando un elemento captura un puntero mediante setPointerCapture().

lostpointercapture: Se dispara cuando se libera un puntero capturado.

pointercancel: Se activa cuando se cancela un evento de puntero.

pointerdown: Se activa cuando un puntero se activa.

pointerenter: Se activa cuando un puntero se mueve a los límites de la prueba de acierto de un elemento o uno de sus descendientes.

pointerleave: Se activa cuando un puntero se mueve fuera de los límites de la prueba de acierto de un elemento.

pointerlockchange: Se activa cuando el puntero está bloqueado/desbloqueado.

pointerlockerror: Se disparó al bloquear el puntero falló.

pointermove: Se activa cuando un puntero cambia de coordenadas.

pointerout: Se activa cuando un puntero se mueve fuera de los límites de la prueba de acierto de un elemento (entre otras razones).

pointerover: Se activa cuando un puntero se mueve a los límites de prueba de acierto de un elemento.

pointerup: Se activa cuando un puntero ya no está activo.

Eventos de selección

selectionchange: Se activa cuando se cambia la selección de texto actual en un documento.

selectstart: Se activa cuando el usuario comienza una nueva selección.

Eventos tactiles

touchcancel: Se activa cuando uno o más puntos de contacto se han interrumpido de una manera específica de la implementación (por ejemplo, se crean demasiados puntos de contacto).

touchend: Se dispara cuando se retiran uno o más puntos de contacto de la superficie táctil.

touchmove: Se dispara cuando uno o más puntos de contacto se mueven a lo largo de la superficie táctil.

touchstart: Se dispara cuando se colocan uno o más puntos de contacto en la superficie táctil.

Clase Element

Es la clase base más general de la que heredan todos los objetos element (es decir, objetos que representan elementos) de un documento. Sólo tiene métodos y propiedades comunes a todo tipo de elementos.

Propiedades

Element también hereda de las interfaces Node y EventTarget.

A continuación se nombraran algunas de las propiedades

Element.attributes: Devuelve un objeto NamedNodeMap que contiene los atributos asignados del elemento HTML correspondiente.

Element.childElementCount: Devuelve el número de elementos secundarios de este elemento.

Element.children: Devuelve los elementos secundarios de este elemento.

Element.classList: Devuelve un objeto DOMTokenList que contiene la lista de atributos de clase.

Element.className: Es un DOMString que representa la clase del elemento.

Element.firstElementChild: Devuelve el primer elemento secundario de este elemento.

Element.id: Es un DOMString que representa el identificador del elemento.

Element.innerHTML: Es un DOMString que representa el marcado del contenido del elemento.

Element.lastElementChild: Devuelve el último elemento secundario de este elemento.

Element.nextElementSibling: Es un elemento, el elemento inmediatamente posterior al dado en el árbol, o si no hay un nodo hermano.null

Element.outerHTML: Es un DOMString que representa el marcado del elemento, incluido su contenido. Cuando se utiliza como setter, reemplaza el elemento con nodos analizados a partir de la cadena dada.

Element.previousElementSibling: Es un elemento, el elemento inmediatamente anterior al dado en el árbol, o si no hay ningún elemento hermano null.

Element.scrollTop: Un número que representa el número de píxeles de la parte superior del documento se desplaza verticalmente.

Controladores de eventos

Element.onfullscreenchange: Un controlador de eventos para el evento fullscreenchange, que se envía cuando el elemento entra o sale del modo de pantalla completa. Esto se puede usar para observar las transiciones esperadas correctas, pero también para observar los cambios inesperados, como cuando la aplicación se ejecuta en segundo plano.

Element.onfullscreenerror: Un controlador de eventos para el evento fullscreenerror, que se envía cuando se produce un error al intentar cambiar al modo de pantalla completa.

Métodos

Al igual que las propiedades Element hereda metodos de su interfaz principal, Node, y por extensión, del elemento primario de esa interfaz, EventTarget.

A continuación se nombraran algunos de los metodos

EventTarget.addEventListener(): Registra un controlador de eventos en un tipo de evento específico del elemento.

Element.after(): Inserta un conjunto de objetos Node o DOMString en la lista de hijos del elemento primario, justo después del archivo.

Element.append(): Inserta un conjunto de objetos Node u objetos DOMString después del último elemento secundario.

Element.before(): Inserta un conjunto de objetos Node o DOMString en la lista de hijos del padre, justo antes del archivo.

Element.closest(): Devuelve el elemento que es el antepasado más cercano del elemento actual (o el elemento actual en sí) que coincide con los selectores dados en el parámetro.

Element.getAttribute(): Recupera el valor del atributo con nombre del nodo actual y lo devuelve como objeto.

Element.getBoundingClientRect(): Devuelve el tamaño de un elemento y su posición en relación con la ventana gráfica.

Element.getElementsByClassName(): Devuelve un archivo HTMLCollection activo que contiene todos los descendientes del elemento actual que poseen la lista de clases proporcionada en el parámetro.

Element.getElementsByTagName(): Devuelve una htmlcollection activa que contiene todos los elementos descendientes, de un nombre de etiqueta determinado, del elemento actual.

Element.hasAttribute(): Devuelve un valor booleano que indica si el elemento tiene el atributo especificado o no.

Element.insertAdjacentElement(): Inserta un nodo de elemento dado en una posición dada en relación con el elemento sobre el que se invoca.

Element.insertAdjacentHTML(): Analiza el texto como HTML o XML e inserta los nodos resultantes en el árbol en la posición dada.

Element.insertAdjacentText(): Inserta un nodo de texto dado en una posición determinada en relación con el elemento sobre el que se invoca.

Element.matches(): Devuelve un valor booleano que indica si la cadena selectora especificada seleccionará o no el elemento.

Element.prepend(): Inserta un conjunto de objetos Node u objetos DOMString antes del primer elemento secundario.

Element.querySelector(): Devuelve el primer nodo que coincide con la cadena selectora especificada en relación con el elemento.

Element.querySelectorAll(): Devuelve una NodeList de nodos que coinciden con la cadena selectora especificada en relación con el elemento.

Element.remove(): Quita el elemento de la lista de elementos secundarios de su elemento primario.

Element.removeAttribute(): Quita el atributo con nombre del nodo actual.

EventTarget.removeEventListener(): Quita un agente de escucha de eventos del elemento.

Element.replaceChildren(): Reemplaza los elementos secundarios existentes de un nodo por un nuevo conjunto especificado de elementos secundarios.

Element.scrollTo(): Se desplaza a un conjunto particular de coordenadas dentro de un elemento determinado.

Element.setAttribute(): Establece el valor de un atributo con nombre del nodo actual.

Element.toggleAttribute(): Alterna un atributo booleano, quitándolo si está presente y agregándolo si no está presente, en el elemento especificado.

dragend: Se activa cuando finaliza una operación de arrastre (soltando un botón del ratón o pulsando la tecla de escape).

dragenter: Se activa cuando un elemento arrastrado o una selección de texto entra en un destino de succión válido.

dragleave: Se activa cuando un elemento arrastrado o una selección de texto deja un destino de entrega válido.

dragover: Se activa cuando se arrastra un elemento o una selección de texto sobre un destino de entrega válido (cada pocos cientos de milisegundos).

dragstart: Se activa cuando el usuario comienza a arrastrar un elemento o una selección de texto.

drop: Se activa cuando se deja caer un elemento o una selección de texto en un destino de entrega válido.

Eventos del raton

auxclick: Se activa cuando se ha presionado y liberado un botón de dispositivo señalador no primario (por ejemplo, cualquier botón del mouse que no sea el botón izquierdo) en un elemento.

click: Se activa cuando se presiona y suelta un botón del dispositivo señalador (por ejemplo, el botón principal de un mouse) en un solo elemento.

dblclick: Se activa cuando se hace clic dos veces en un botón de dispositivo señalador (por ejemplo, el botón principal de un mouse) en un solo elemento.

mousedown: Se activa cuando se presiona un botón de dispositivo señalador en un elemento.

mouseenter: Se activa cuando un dispositivo señalador (generalmente un mouse) se mueve sobre el elemento que tiene el oyente conectado.

mouseleave: Se activa cuando el puntero de un dispositivo señalador (generalmente un mouse) se mueve fuera de un elemento que tiene el oyente conectado a él.

mousemove: Se activa cuando un dispositivo señalador (generalmente un mouse) se mueve mientras está sobre un elemento.

mouseout: Se activa cuando un dispositivo señalador (generalmente un mouse) se mueve fuera del elemento al que está conectado el oyente o de uno de sus hijos.

mouseover: Se activa cuando un dispositivo señalador se mueve sobre el elemento al que está conectado el oyente o sobre uno de sus hijos.

mouseup: Se activa cuando se suelta un botón de dispositivo señalador en un elemento.

webkitmouseforcechanged: Se dispara cada vez que la cantidad de presión cambia en la pantalla táctil del trackpad.

webkitmouseforcedown: Se dispara después del evento de ratón hacia abajo tan pronto como se haya aplicado suficiente presión para calificar como un "clic de fuerza".

webkitmouseforcewillbegin: Se ha disparado antes del evento mousedown.

webkitmouseforceup: Se dispara después del evento webkitmouseforcedown tan pronto como la presión se ha reducido lo suficiente como para finalizar el "clic de fuerza".

Eventos tactiles

touchcancel: Se activa cuando uno o más puntos de contacto se han interrumpido de una manera específica de la implementación (por ejemplo, se crean demasiados puntos de contacto).

touchend: Se dispara cuando se retiran uno o más puntos de contacto de la superficie táctil.

touchmove: Se dispara cuando uno o más puntos de contacto se mueven a lo largo de la superficie táctil.

touchstart: Se dispara cuando se colocan uno o más puntos de contacto en la superficie táctil.
